# 博客文章写作指南

本文档详细说明论文转博客的写作规范、要求和最佳实践。

## 论文理解检查清单

在开始写作前，确保理解以下内容：

### 1. 问题与动机
- [ ] 论文要解决什么问题？
- [ ] 这个问题为什么重要？
- [ ] 现有方法的局限是什么？
- [ ] 论文的切入点是什么？

### 2. 核心方法
- [ ] 论文的核心思想是什么？
- [ ] 包含哪些关键模块？
- [ ] 每个模块的输入、变换、输出是什么？
- [ ] 方法的创新点在哪里？
- [ ] 训练或推理流程是怎样的？

### 3. 实验部分
- [ ] 使用了哪些数据集？
- [ ] Baseline 是什么？
- [ ] 评估指标有哪些？
- [ ] 主要实验结果是什么？
- [ ] 有哪些消融实验？说明了什么？
- [ ] 有哪些失败案例或负面结果？

### 4. 讨论部分
- [ ] 论文的主要贡献是什么？
- [ ] 方法的局限性是什么？
- [ ] 适用边界在哪里？
- [ ] 对后续研究或应用有什么启发？

### 5. 章节映射
- [ ] 列出论文的所有章节标题和层级
- [ ] 为每个章节规划对应的博客内容位置
- [ ] 标记需要重点解读的章节

### 6. 术语
- [ ] 论文中有哪些关键术语需要解释？
- [ ] 公式是否正确理解？

对不确定的内容明确标注"论文未明确说明"。

## 章节引用规范

所有来自论文的观点、数据、结论，必须标注对应的原论文章节号，格式为【§X.X】即可。

### 章节覆盖要求

1. 写作前先梳理出论文的所有章节标题，形成章节清单
2. 为每个章节规划对应的博客内容位置，确保没有遗漏
3. 对于简短的章节（如致谢、参考文献），可以合并到适当的位置
4. 写作完成后对照章节清单检查，标记每个章节的覆盖情况

## Frontmatter 详细说明

### title

使用中文标题，清晰表达论文主题。

**示例：**
- "FSDP简介"
- "代码走读：FSDP叠加TorchAO Float8低精训练"
- "论文解读：XXX方法"

### author

固定为 "Tong"。

### date

使用格式 "YYYY-MM-DD HH:MM"，例如 "2026-03-17 16:30"。

### updated

如果有更新，填写更新时间；如果没有，与 date 相同。

### tags

根据论文内容选择相关标签，例如：
- AI / LLM / Transformers
- PyTorch / Training / Inference
- 具体技术领域标签

### categories

固定为 "- 技术积累"。

## 正文各部分写作指南

### 概述

- 开门见山，说明论文解决什么问题
- 用 2-3 句话总结核心贡献
- 让读者快速知道这篇文章讲什么

### 背景知识（可选）

- 讲解必要的前置知识
- 说明为什么这个问题重要
- 分析现有方法的局限
- 引出论文的切入点

### 核心方法

- 先讲整体思路
- 再拆解关键模块
- 每个模块讲清楚：输入、变换、输出、设计理由
- 使用 mermaid 图表辅助说明，1-3 张关键图即可
- 流程图使用 flowchart，时序图使用 sequenceDiagram

### 实验与结果

- 说明实验设置：数据集、Baseline、评估指标
- 展示主要结果，用清晰的方式呈现
- 分析结果说明了什么
- 介绍消融实验（如有）

### 讨论

- 总结论文亮点：什么是这篇论文最有价值的地方
- 说明局限性：论文没有解决什么，适用边界在哪里
- 探讨启发：对后续研究或实际应用有什么指导意义

### 参考资料

- 不需要重复列出论文原文链接（博客头部信息已经提供）
- 只需要补充相关的延伸资源链接（如论文引用的关键工作、用到的开源项目等）
- 使用 Markdown 链接格式，每个链接补充简短说明，说明该资源的用途

### 标准结尾

必须包含以下内容：

```markdown
---

> 本文采用了AI生成的文本，并全部经过人工审核编辑。
```

## 写作风格要求

### 语言

- 使用中文表达
- 自然、准确、深入浅出
- 避免"AI味"空话
- 技术讲解要清晰，适合博客读者

### 公式

使用 LaTeX 语法：

```markdown
$$
Y = XW^\top \quad \text{其中} \quad Y \in \mathbb{R}^{B \times L \times D_{out}}
$$
```

行内公式使用单个 `$`：

```markdown
权重矩阵 $W$ 的 shape 为 $(D_{out}, D_{in})$
```

### 代码块

添加语法高亮：

```python
def example_function():
    """这是一个示例函数"""
    return True
```

### 引用

- 对不确定的内容明确标注"论文未明确说明"
- 引用论文中的内容时，说明来源

## 图片处理

### 图片文件夹

创建同名文件夹存放图片：

```
source/_posts/
├── my-blog-post.md
└── my-blog-post/
    ├── img1.jpg
    ├── img2.jpg
    └── img3.jpg
```

### 图片引用

使用相对路径引用：

```markdown
![图片描述](./img1.jpg)
```

### 图片说明

可以在图片下方添加说明：

```markdown
![](./img1.jpg)
*图片来源：论文原文 Figure 1*
```

### 兜底规则
如果没有成功获取到论文的配图资源，直接跳过配图部分即可，不需要在博客中专门说明，也不需要强行添加无关图片。

## 博客风格参考

### 教程式风格

**适用场景：** 介绍新概念、新技术

**结构特点：**
- 从简单概念逐步深入
- 包含完整的代码示例
- 大量图片辅助理解
- 清晰的表格展示对比

**建议结构：**
1. 概念回顾（前置知识）
2. 新技术介绍
3. 使用示例（含完整代码）
4. 总结

### 代码走读风格

**适用场景：** 分析代码实现、技术细节

**结构特点：**
- 深入分析实现细节
- mermaid 流程图展示流程
- 关键代码位置索引
- FAQ 解答常见疑问

**建议结构：**
1. 概述
2. 背景知识
3. 核心机制分析
4. 实现细节
5. 关键代码位置索引
6. FAQ

## Slug 与文件名生成规则

### Slug 生成规则
根据论文标题生成 slug：

1. 提取论文标题的核心关键词
2. 转换为小写
3. 使用连字符连接单词
4. 去掉特殊字符（标点、符号等）
5. 保持简洁，通常 5-10 个单词

**示例：**
- "Attention Is All You Need" → "attention-is-all-you-need"
- "BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding" → "bert-pre-training-transformers"
- "PyTorch FSDP: Experiences on Scaling Fully Sharded Data Parallel" → "pytorch-fsdp-scaling"

### 文件名规则
博客文件直接使用 `{slug}.md` 作为文件名，不需要添加 `YYYY-MM-DD-` 时间戳前缀。
图片目录与 slug 同名，与博客文件同级存放。

## 交付格式

博客生成后，使用以下格式告知用户：

```
✅ 博客文章已生成！

📄 文件：source/_posts/{slug}.md
```

如果创建了图片文件夹，补充说明：

```
✅ 博客文章已生成！

📄 文件：source/_posts/{slug}.md
🖼️ 图片：source/_posts/{slug}/ 文件夹
```
