# 翻译与编辑偏好（译者定稿风格）

以下偏好从译者人工修改 AI 初稿的过程中提炼。翻译时主动遵循，可显著减少返工。

## 译文风格

- **意译优先，拒绝逐字直译（calque）。** 重写句子使其符合中文表达习惯，不要保留英文句式（长定语从句、破折号串联等）。
- **拆分长句。** 原文用破折号（——）串起来的长句，倾向拆成几个短句，提升可读性。
- **全角中文标点。** 正文一律用全角标点（，。、：""），不混用半角。
- **保持小节结构清晰。** 原文把不同主题揉在同一节时，可拆出独立 `##` 小标题（如把「插件/连接器」从 Skill 节里拆出）。
- **引言段单独提为「## 摘要」小节。** 原文开头的概述段，不直接作正文首段，而是用 `## 摘要` 小标题单独成节，与正文 `---` 分隔。
- **副标题可并入主标题。** 原文有副标题时，可用「：」把副标题并入 H1 主标题（如「从技术债务到认知债务与意图债务：在 AI 时代重新思考软件健康」），frontmatter `title` 用同样写法。也可仍用 `####` 副标题另起，二者择一即可。

## 译者增值内容

- **译者按里加原文链接。**
  - 一般博文：单独一行写「原文链接：[URL](URL)」。
  - **学术博文用完整学术引用格式作链接文本**：作者、年份、标题、刊名卷期、DOI 全写，斜体刊名用 `*...*`，整体作为超链接指向原文 URL。例：`[Storey, M.-A. 2026. From Technical Debt to Cognitive and Intent Debt: Rethinking software health in the age of AI. *ACM Queue* 24, 2 (June 2026). DOI: https://doi.org/10.1145/3807966](https://spawn-queue.acm.org/doi/10.1145/3807966)`。
- **译者按可加编者评论。** 不只概述原文，可补一两句面向中文读者的意义解读。
- **用词克制。** 不夸大（"一篇重要文章" → "一篇文章"），平实表述。
- **为陌生专有名词加译者注。** 人名/产品/概念首次出现时，用「（译者注：…）」补身份或背景（如 "Peter Steinberger（译者注：…创办人）"）。

## 链接处理

- **保留原文全部链接，包括产品文档链接。** 不只保留作者博客内链；Codex Automations / Skills / Subagents 这类官方文档链接也要在对应位置保留。
- **引用类链接（推文等）改挂到自然的中文短语上。** 原文挂在某个英文单词（careful / said / loop）上的引用链接，译文中挂到通顺的中文短语，而不是逐字硬挂。

## 学术文献引用（带参考文献列表的博文）

当原文有上标文献编号（如 `...24`）且文末附参考文献列表时，按此处理：

- **参考文献列表全文照搬不译**：条目内的英文作者/标题/出处/期刊名一律不译，仅 URL 保留；可译的只有连接词与「Available at」之类。给每条加锚点 `<a id="ref-N"></a>` 放在行首，与正文引用对应。
- **正文引用改为可点的斜体超链接，指向文末对应条目**，不保留无链接的纯上标数字：
  - 单条：`[*[N]*](#ref-N)` —— 超链接里套斜体的方括号数字。
  - 多条：`[[a](#ref-a),[b](#ref-b)]` —— 每个数字各自可点。
- **图也可交叉引用**：图前加 `<a id="ref-img-N"></a>` 锚点，正文用「见[*图 N*](#ref-img-N)」链接过去。

## 术语偏好（覆盖默认译法）

| 默认 / 直译 | 译者偏好 |
|---|---|
| 构件 | **组件** |
| 任务分级收件箱 / 收件箱 | **任务分类邮箱 / 分类邮箱** |
| 校验者（maker/checker） | **执行者 / 验证者** |
| prompt（小写） | **Prompt / Agent** 倾向首字母大写作专有名词 |

- 保留概念词/工具名的英文括注，如 钩子（Hooks）、编排开销、意图债务、认知债务、认知屈服。
- **artifact 的译法按当次术语表定夺**：可译「产出/制品」（通俗），也可直接保留英文 `artifacts` 不译夹在中文里。一篇内须统一；同篇内不要「产出/制品/artifacts」混用。
- 同一概念在不同文章译法可能不同（如 comprehension debt 在 loop-engineering.md 译「认知债务」，在本篇译「理解债务」），以当次 `terms.txt` 为准，且须在译文里点明与相关概念的层次区别。
- 其余术语以当次的术语对照表（`terms.txt`）定稿为准。

## Frontmatter 偏好

- `author` 用**半角括号 + 空格**：`原作者名 (Tong 译)`（不要用全角 （））。
- 初次发布后若修订，加 `updated:` 字段（日期 + 时间）。
